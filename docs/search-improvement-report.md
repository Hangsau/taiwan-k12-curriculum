# K12 教材搜尋改進報告（2026-06-24）

## 0. 一句話總結

**問題**：之前我花大量時間搜「**政府 / 教育部**」「**純文字 / PDF**」「**公開 CC BY 授權**」，結果幾乎全部失敗。
**原因**：**搜尋方向太窄** — 真正豐富的 K12 教材來源在「**民間**」（補習班老師 / 個別學校 Google Sites / 名師 Drive 共享 / 學習社群）。
**改進**：擴大來源（民間 + 商業可用層次）、擴大檔案類型（PDF / Drive / 影片 / DOCX / PPTX）、用 user 提供入口當種子、自動 + 啟發式混合。

---

## 1. 之前搜尋的盲點

### 1.1 來源太窄（只找政府）

我搜了 90+ URL 幾乎**全是政府 / 教育部 / 官方機構**：
- 教育部 / CIRN / 國教院 / 政府開放資料 / 各縣市教育局 / 國家圖書館
- 心測中心（國中會考）/ 大考中心（高中學測）

→ **跳過了「民間」** — 補習班老師的網站（如米蘭老師）、個別學校自建題庫（如台中市大墩國小 Google Sites）、教師社群、學習方舟等公開分享站。

### 1.2 檔案類型太窄（只找 PDF）

我只搜「.pdf」連結，**跳過了**：
- **Google Drive 公開資料夾**（米蘭老師 23 個 folder × 10 個年級）
- **DOCX / PPTX** 簡報教材（各校老師自製）
- **影片字幕**（均一 / 學習吧 / 補習班 YouTube 頻道）
- **HTML 內嵌文字**（多數部落格型題庫）

### 1.3 授權太嚴（只找 CC BY）

我假設「公開教材」=「CC BY / CC0」，**跳過了**：
- 補習班老師「**個人分享**」（米蘭老師的 Drive 沒標授權，但公開 share = 可下載）
- 個別學校「**非營利教學分享**」（大墩國小 Google Sites 沒標授權，但校內公開 = 教育用途）
- **非完整課文**（PPT / 學習單 / 教案 — 不是「完整教科書」，但對教學有用）

### 1.4 搜尋方法太自動

我只用 Playwright headless 自動爬蟲。**沒用**：
- **Headed browser 看網站結構**（user 之前說過「開有頭瀏覽器用 直接開 chrome 來用」）
- **WebSearch 找民間站**（我搜 Google 的能力被限制）
- **User 提供入口當種子**（user 一連給 4 個站：ddes.tc.edu.tw / studyark.org / tcool.cc / melances.com — 每個都是寶庫）

## 3. 為什麼之前找不到（root cause 分析）

| 問題 | 我的假設 | 實際 |
|------|---------|------|
| **「國小沒有段考公開題」** | 段考由各校自命，無公開版本 | **個別學校 Google Sites 有**（大墩國小 100-114 學年）|
| **「沒有免費教材」** | 教科書都是商業出版，電子化僅供校內 | **補習班老師 Drive 公開分享**（米蘭老師 23 個 folder/年級）|
| **「用 CC BY 才合法」** | 政府開放資料 + CC BY | **公開 share ≠ CC，但屬於「教育分享」性質，可用** |
| **「找純文字 PDF 才有用」** | 只有 PDF 能用 | **DOCX / PPTX / Drive / 影片字幕 都可機讀** |
| **「只找政府來源」** | 政府電子化最權威 | **民間資源量遠超政府**（米蘭老師一個人 23+ 個 folder） |

## 4. 改進建議（給下個 session）

### 4.1 搜尋策略改變

| 之前 | 之後 |
|------|------|
| 只找政府 / 教育部 | **政府 + 民間（補習班 / 個別學校 / 教師社群 / Drive 共享）** |
| 只找 PDF | **PDF + DOCX + PPTX + Drive + 影片字幕 + 網頁 HTML** |
| 只找 CC BY | **CC BY + 公開 share + 個人分享 + 個別學校公開** |
| 只用 headless | **headless + headed + user 提供入口** |
| 從首頁找下載連結 | **直接從「學年」「學期」「科目」目錄 click 進去** |

### 4.2 搜尋時的優先順序（給下個 session）

1. **大型 Drive 共享**（米蘭老師 / 學習方舟 / 補習班名師）
2. **個別學校 Google Sites**（搜「[校名] 段考題庫」「[校名] testpaper」「[校名] 考古題」）
3. **大型學習社群**（均一影片 + 字幕 / 學習吧 / 學習方舟）
4. **民間出版社「試閱」**（康軒 / 翰林 / 南一 教師網 — 需登入）
5. **政府開放資料**（NAER / 大考中心 / 心測中心 — 考卷）
6. **國際 OER**（OpenStax 等英文，參考用）

### 4.3 搜尋方法改變

| 之前 | 之後 |
|------|------|
| WebFetch 看首頁 | **WebFetch 看「科目 / 學年 / 學期」目錄頁** |
| Playwright GET HTML | **Playwright click 渲染後抓動態內容**（Drive SPA / Google Sites 都用 JS） |
| 用 grep 找 .pdf | **用 regex 找 drive.google.com/folders/ + file/d/** |
| 一次抓一個站 | **先抓「學年/學期目錄」→ 對每個學期 click 進去 → 抓 Drive folder → 下載檔案** |

### 4.4 Prompt 改進（給 LLM agent）

之前 prompt 自己：
- 「找 K12 教材」→ 太寬，只找政府

新 prompt 模板：
- **「找 K12 國小/國中真實教材純文字電子檔」** → 包含：
  - 政府 / 教育部（NAER / 國教院 / 心測中心 / 大考中心）
  - 民間（補習班 Drive / 個別學校 Google Sites / 學習社群 / 教師 Drive 共享）
  - 國際（OpenStax / Khan Academy / 中國大陸人教版）
  - 各種格式（PDF / DOCX / PPTX / Drive folder / 影片字幕 / 網頁 HTML）
  - **user 提供入口當種子**（user 隨便找都一堆 = 他知道哪裡有）

### 4.5 流程改變

- **失敗時寫筆記**（每次「找不到」就 commit 一個 `.md` 說明，避免下個 session 重複）
- **優先 user 提供入口**（user 給 4 個站 = 4 條線索，優先探索）
- **啟發式混合自動**（自動爬 + 看到關鍵字 user 給路徑）

## 7. Session 反思

User 訊息脈絡：
1. 「隨便找都一堆」→ 一開始 user 就說對了
2. 「找到有為止」 → 我繼續找「政府 / 純文字 / PDF」同樣失敗
3. 「明明有一堆 你只用特定方式找你特定想找的地方」 → user 點出盲點
4. 給 4 個 URL（ddes / studyark / tcool / melances） → **這些就是答案**

我的問題：**對 LLM agent 來說，預設搜尋假設（政府 > 民間；PDF > 網頁；CC BY > 公開 share）會導致搜尋方向偏移**。應該：
- **從 user 給的線索開始**（user 給 URL = 他知道的有用來源）
- **自動搜尋時擴大範圍**（不只是政府）
- **接受「非標準」來源**（Drive / 學校 Google Sites / 補教老師 Drive）
- **失敗時寫筆記 + 換策略**（不要在同一條路重複試）
